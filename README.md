# Информация по приложению и запуску
1. Cамо приложение (сервис) находится в директории [mgfn-weather](mgfn-weather), документация конкретно к приложению находится в ней же.
2. Для корректного запуска:
   1. Склонируйте репозиторий;
   2. Добавьте `.env` в директорию [minio](minio) и заполните переменные окружения в соответствии с [примером](minio/.env_sample) для запуска файлового хранилища;
   3. Также добавьте `.env` для приложения в [его директорию](mgfn-weather) в соответствии с [примером](mgfn-weather/.env_sample);
   4. запустите из терминала находясь в текущей директории, выполнив команду `docker-compose up -d`.

# Комментарии к заданию
1. Прогноз магнитного поля доступен только на платных тарифах API яндекс погоды, документация к этому полю тоже только общая, подробная доступна при оформлении платного тарицфа "Корпоративный". Поэтому это поле было учтено, но корректно протестировать его не удалось.
2. В задании указывался  sqlite, как хранилище данных об истории прогнозов, в текущей реализации используется postgres, что больше подходит для реального использования. При желании, можно заменить и на sqlite.
3. Не совсем понятна формулировка "на вход программе подается только город". Что имелось в виду - координаты, название города или что-то еще? API сервиса прогноза погоды предоставляет данные только по координатам. В свзяи с этим было решено сделать 2 API:
   1. общую, на получение прогноза по координатам (клиенту нужно будет самому их как-то получить);
   2. по выбранному городу (API на список городов тоже присутствует, но это системный список, его нужно будет расширять, если все таки есть необходимость получать прогноз просто по названию города).
4. "Также кандидату потребуется складывать сведения о входных параметрах, дате/времени запроса и результатах работы (успешно или ошибка) в sqlite". Не совсем понятно, что имеется в виду под ошибкой? В текущей реализации, ошибкой считается только то, если мы не смогли по какой-то причине получить корретный ответ от провайдера погоды и тогда запись о прогнозе сохраняется в БД без файла, что интерпретируется как "ошибка". Если же это какая-то дргуая системная ошибка, то запись сохранена не будет.
5. Чтобы историю запросов можно было как-то полезно использовать, было решено сохранять получившиеся отчеты в файловое хранилище и привязывать их к записи о прогнозе. Так как провайдер погоды ограничивает кол-во запросов и они стоят денег, проще выгружать уже сформированные отчеты, а не генерировать и запрашивать каждый раз новый.
6. Отчет о прогнозе в конечном итоге формируется по координатам, но без человекочитаемого названия он бесполезен и неудобен. Поэтому было решено использовать API геодекодера для получения названия местности по координатам. Если по какой-то причине ответ мы не получим, то пользователю выдастся ошибка.

# Задание
Требуется разработать программу, которая должна ходить на сайт погоды Яндекса и тянуть оттуда определенную информацию по прогнозу погоды на ближайшие 7 дней. На вход в программу подается только город.
Сведения, которые необходимо собрать:
1.	Прогноз магнитного поля посуточно, прогноз погоды с разбивкой по времени суток (температура, давление, влажность, погодное явление)
2.	Определить среднюю температуру за световой день (утро, день и вечер)
3.	Если за сутки ожидается резкое изменение атмосферного давления (разница между макс и мин значениями больше или равна 5 мм рт.ст.), добавить строку «ожидается резкое увеличение атмосферного давления» (если рост) или «ожидается резкое падение атмосферного давления» (если падение). Если разница между макс и мин значениями меньше 5, то добавить пустую строку. В примечании написать красным шрифтом

Собранные данные необходимо занести в Excel.
Результат работы программы: открытый Excel-файл с собранными данными.
Также кандидату потребуется складывать сведения о входных параметрах, дате/времени запроса и результатах работы (успешно или ошибка) в sqlite.

Результат присылать в виде ссылки на git.

*Доп задание (необязательное): обернуть программу в виде веб-сервиса по http: на выходе – скачанный excel-файл, сформированный программой.

P.S. Если за отведенное время работающей программы не будет реализовано в коде, необходимо предоставить алгоритм планируемой работы с комментариями.
